# ClaudeDesktopPlusPlus 开发追踪

更新时间：2026-07-06

## 产品定位

ClaudeDesktopPlusPlus 现阶段不再主打“推荐页”或商业广告位，而是先把产品打磨成一个可信的 Claude Desktop 诊断、修复、配置和启动控制台。

核心定位：

> Claude++ 是 Claude Desktop Doctor & Launcher。它帮助用户诊断 Claude Desktop 为什么不能用，修复本地环境和历史数据，配置第三方 API，并一键启动增强后的 Claude Desktop。

## 当前产品决策

| 决策 | 状态 | 说明 |
| --- | --- | --- |
| 暂停推荐页 | 已确认 | 应用内暂不突出赞助商或商业推荐，避免过早暴露商业目的。README 可保留赞助鸣谢，但主产品体验以诊断和修复为中心。 |
| 首页改为体检报告 | 待做 | 用户打开应用后第一眼看到 Claude Desktop、系统环境、API、插件、汉化和历史状态。 |
| 保留 API 配置能力 | 进行中 | API 页面继续支持 cc-switch 同步、手动 Provider、直连/Gateway、凭据测试和模型发现。 |
| 强化一键修复历史 | 进行中 | 历史修复保留简单入口，默认自动选择推荐来源并先备份。 |
| 学习 Codex++ 双入口 | 待做 | 后续拆分为静默启动入口 `Claude++` 和管理入口 `Claude++ 控制台`。 |
| Claude Desktop 内部状态菜单 | 待做 | 通过外部 launcher + CDP 注入，在 Claude Desktop 内显示 Claude++ 状态和控制入口。 |

## 阶段计划

### Phase 1：重新建立产品亮点

目标：让用户一打开软件就知道它解决什么问题。

| 任务 | 状态 | 验收标准 |
| --- | --- | --- |
| 首页改为 Claude 体检报告 | 待做 | 首页展示安装、系统、API、插件、汉化、历史 6 类状态。 |
| 首页提供一键修复并启动 | 待做 | 一个主按钮根据诊断结果执行推荐动作或引导用户到对应页面。 |
| 推荐页从主导航移除或隐藏 | 待做 | 默认导航不出现“推荐”，商业内容不进入核心操作路径。 |
| 关于页保留项目说明和赞助鸣谢 | 待做 | 商业信息只作为项目支持信息出现，不打断主要使用流程。 |
| 诊断结果支持复制 | 待做 | 用户可复制脱敏后的诊断日志用于 issue 反馈。 |

### Phase 2：API 接入诊断器

目标：不是只让用户填配置，而是告诉用户为什么不能用。

| 任务 | 状态 | 验收标准 |
| --- | --- | --- |
| cc-switch 完整同步与去重 | 进行中 | 同步后列表与 cc-switch 当前配置一致，不保留已删除残留项。 |
| Provider 删除、编辑、设为当前 | 已完成 | 手动新增配置可删除，列表不会无限膨胀。 |
| 凭据测试弹窗反馈 | 已完成 | 点击测试后在当前操作位置直接显示结果，不要求用户回到页面顶部。 |
| 模型列表自动获取 | 进行中 | 根据 Base URL 和 Key 自动拉取可用模型。 |
| 协议识别 | 待做 | 自动判断 Anthropic 兼容、OpenAI/Codex 兼容或未知协议。 |
| 直连可用性判断 | 待做 | 明确提示能否直连，以及是否缺少 Claude 模型映射。 |
| Gateway 推荐策略 | 待做 | 当上游不支持直连映射时，给出 Gateway 建议和影响说明。 |

## 近期优先级

1. 首页改为 Claude 体检报告。
2. 从主导航隐藏推荐页，把赞助信息移到关于页或 README。
3. API 配置页补齐协议识别、直连可用性判断和 Gateway 建议。
4. 系统就绪页补充 VMP 失败原因解释和诊断日志复制。
5. 设计 Claude Desktop 内部状态菜单的最小可行版本。

## B 站宣传主线

当前宣传不强调商业推荐，重点讲用户痛点：

1. Claude Desktop 配置复杂。
2. 第三方 API 明明有 Key，但模型不显示。
3. 直连和 Gateway 不知道怎么选。
4. 切换账号或渠道后历史可能丢失。
5. 插件、汉化、系统环境都需要到处找资料。

宣传口径：

> Claude++ 不是单纯的 API 配置器，而是一个 Claude Desktop Doctor。它会帮你检查问题、给出修复建议，并一键启动配置好的 Claude Desktop。

## 风险与边界

| 风险 | 处理方式 |
| --- | --- |
| 过早商业化导致用户不信任 | 推荐页暂停，商业内容只放在 README 或关于页的鸣谢区域。 |
| 用户误解为绕过官方权益 | 文案强调本工具做本地配置、诊断和增强启动，不伪造账号或承诺官方权益。 |
| Claude Desktop 更新导致注入失效 | 注入功能必须有健康检查、失败提示和版本兼容记录。 |
| Gateway 能用但体验受限 | UI 明确说明直连/Gateway 差异，并提示模型映射要求。 |
| 系统修复失败难以解释 | 保留原始日志，提供可复制诊断报告。 |
